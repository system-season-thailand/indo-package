# Season B2B Lab — حزمة قائمة بذاتها (Standalone Lab)

حزمة اختبار كاملة تُرفع على Netlify وتعمل **بالكامل ببيانات تجريبية**، دون أي اتصال بقاعدة بيانات
أو مفاتيح إنتاج. نقطة الدخول **بوابة العمليات**، ومنها تُفتح كل الوحدات.

## التشغيل / الرفع

1. **محلياً:** افتح `operations-portal.html` (أو `index.html` الذي يحوّل إليها).
2. **Netlify:** اسحب مجلد الحزمة كاملاً إلى Netlify Drop؛ جذر الموقع `/` يفتح البوابة تلقائياً
   (عبر `index.html` و`_redirects`). كل الملفات في **مجلد واحد مسطّح** → المسارات النسبية تعمل بلا 404.

## الوحدات (١٠ وحدات × ٥ ملفات = ٥٠ ملفًا + ملفات تشغيل)

- **بوابة العمليات (Shell):** `operations-portal.*` + `operations-portal-sample-data.js` + ملاحظات المعمارية.
- **لوحة الإدارة:** `management-dashboard.*` + بياناتها + ملاحظاتها.
- **حالات العروض:** `quotation-status-manager.*` + بياناتها + ملاحظاتها.
- **الحجوزات المؤكّدة:** `confirmed-bookings-manager.*` + بياناتها + ملاحظاتها.
- **مسار الحجوزات:** `booking-pipeline.*` + بياناتها + ملاحظاتها.
- **إدارة الشركات (CRM):** `companies-management.*` + بياناتها + ملاحظاتها.
- **مساحة العمليات (الكوكبيت الموحّد):** `operations-workspace.*` + بياناتها + ملاحظاتها.
- **مختبر المستندات (Document Lab):** `voucher-experience-lab.*` + بياناتها + ملاحظاتها — معاينة المستندات الخمسة.
- **استوديو المحتوى (Content Studio):** `content-studio-lab.*` + بياناتها + ملاحظاتها — مصدر الحقيقة للمحتوى.
- **محرّك القوالب (Template Engine):** `premium-template-lab.*` + بياناتها + ملاحظاتها — قالب واحد بثلاث لغات تصميم، يُجمَّع من بلوكات (يستهلك بيانات الاستوديو).
- **تشغيل ومشترك:** `index.html` · `_redirects` · `content-store.js` (جسر مشترك يربط استوديو المحتوى بمحرّك القوالب) · هذا الملف.

## الكيان الرئيسي و دورة الحياة

- **الشركة** هي الكيان الرئيسي، ويُربط كل شيء عبر `company_id`.
- **`booking_id`** هو المفتاح الرئيسي بعد تأكيد العرض، ومنه تُولّد المستندات لاحقاً.

```
الشركة (company_id) → العروض → الحجوزات (booking_id) → الفواتير → التقارير
```

## للمطوّر (بندر) — الربط المستقبلي

ابحث عن العلامات داخل ملفات البيانات/المنطق:
- `SAMPLE DATA SOURCE` — البيانات التجريبية الحالية.
- `FUTURE SUPABASE DATA SOURCE` — مكان استبدالها باستعلام Supabase (بنفس شكل البيانات).
- `DO NOT WRITE IN LAB MODE` — لا كتابة لقاعدة في المختبر.

أبقِ البوابة قشرة موحّدة بنفس بنية التنقّل والأدوار؛ واربط كل الوحدات عبر `company_id` و`booking_id`.

## القيود (مؤكّدة)

لا Supabase · لا مفاتيح إنتاج · لا تعديل على منطق الإنتاج · لا كتابة فعلية · بيانات تجريبية فقط.
