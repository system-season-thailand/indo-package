# Deploying the Chromium PDF pipeline (server-side)

Until one of these is deployed, your site keeps using the old preview/print path.
The renderer (`pdf-render.html` + 5 scripts) is already correct — it just needs a
host that can launch a real Chromium. Pick ONE path below.

The "Generate PDF" button is already wired: open the Template Engine preview →
**«توليد PDF (خادم)»**. It POSTs the saved snapshot and downloads the finished PDF.

---

## Path A — Netlify Function (same site, recommended to try first)

Everything stays on your existing Netlify site. One button, same domain.

**Important:** this CANNOT be a drag-and-drop static deploy. Functions need their
npm deps installed/bundled, so you must deploy via a Git repo or the Netlify CLI.

Files included for this path:
- `netlify/functions/generate-pdf.js`  — the function
- `netlify.toml`                       — functions config
- `package.json`                       — `@sparticuz/chromium` + `playwright-core`
- `pdf-export-client.js`               — already points at `/.netlify/functions/generate-pdf`

### Steps
1. Put this whole folder in a Git repo (GitHub/GitLab).
2. In Netlify → **Site → Build & deploy → Link repository** (or `netlify init`).
3. Netlify reads `netlify.toml`, runs `npm install`, bundles the function.
4. Deploy. The endpoint is `https://<your-site>/.netlify/functions/generate-pdf`.
5. On the site: Template Engine → preview → **«توليد PDF (خادم)»** → PDF downloads.

### Verify it's live (no UI needed)
```
curl -s -X POST https://<your-site>/.netlify/functions/generate-pdf \
  -H "content-type: application/json" \
  -d '{"dest":"bali","style":"magazine"}' -o out.pdf
pdfinfo out.pdf      # expect: Pages: 8, Producer: Skia/PDF
```

### Known limits (be aware)
- **Timeout:** free tier sync functions cap at 10s; image-heavy guides can exceed
  this on a cold start. Pro+ allows up to 26s (uncomment `timeout` in `netlify.toml`).
- **Request body:** the snapshot (with base64 images) must fit Netlify's request
  limit (~6 MB). Large media libraries can exceed it — use Path B if so.
- **Version match:** `@sparticuz/chromium` and `playwright-core` must stay
  compatible (see note in `package.json`). Mismatch = launch failure.

If you hit timeouts or size limits, use Path B.

## Path B — Dedicated Node service (most reliable, no limits)

Run `pdf-server.js` (already in this package, zero npm deps — uses the host's
system Chrome) on any always-on host: Render, Railway, Fly.io, or a small VPS.

### Steps
1. Deploy `pdf-server.js` + the static lab files to the host. Ensure Chrome/Chromium
   is installed; set `CHROME_PATH` to its binary (default `/opt/google/chrome/chrome`).
2. Start it: `node pdf-server.js` (listens on `:8787`). Put it behind HTTPS.
3. On your Netlify site, set the service URL **before** `pdf-export-client.js`:
   ```html
   <script>window.PDF_SERVICE_URL = "https://pdf.yourhost.com";</script>
   <script src="pdf-export-client.js"></script>
   ```
   The client then posts to `https://pdf.yourhost.com/export`.
4. Same button — **«توليد PDF (خادم)»** — now hits your dedicated server.

### Verify
```
curl -s -X POST https://pdf.yourhost.com/export \
  -H "content-type: application/json" \
  -d '{"dest":"bali","style":"magazine"}' -o out.pdf
pdfinfo out.pdf      # expect: Pages: 8
```

This is the engine proven to produce 8 pages / 0 blanks in testing.

---

## What is NOT a valid host
- **Netlify static hosting alone** — no server, can't run Chromium.
- **Supabase Edge Functions** — Deno edge runtime, cannot spawn a Chromium process.
- **iPhone Safari / Share / Print** — abandoned. Not an export path.

## Success criteria (both paths)
From the deployed site, click **«توليد PDF (خادم)»** and receive a PDF that is
**8 pages, no blank pages, with the images from your saved content** — generated by
Chromium server-side, downloaded directly (Safari never renders it).
