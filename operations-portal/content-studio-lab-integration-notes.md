# استوديو المحتوى (Content Studio Lab) — ملاحظات (Phase 14.8)

> **أساس إدارة المحتوى** الذي ستستهلكه كل الأدلة والقسائم والبرامج وحقائب السفر مستقبلاً.
> ليس مشروع دليل وجهة · ليس PDF · ليس توليد مستندات.
> لا Supabase · لا منطق إنتاج · لا رفع فعلي · بيانات تجريبية فقط · تعديلات الجلسة في الذاكرة فقط.

## المبدأ الأساسي

```
استوديو المحتوى → مكتبات المحتوى → محرّك المستندات → دليل / قسيمة / برنامج / حقيبة
```
الوثائق **لا تملك** المحتوى — بل **تستهلكه**. الاستوديو هو مصدر الحقيقة؛ الوثائق مخرجات.

## الأقسام الأربعة

1. **محتوى الوجهات** (٩): بالي · أوبود · سيمينياك · نوسا دوا · أولواتو · بونشاك · بانكوك · بوكيت · المالديف — لكلٍّ: غلاف · معرض · معلومات وصول · شريحة · عملة · تنقّل · طوارئ · توصيات (كلها قابلة للتحرير في الجلسة).
2. **محتوى الفنادق** (٤): اسم · غلاف · معرض · وصف · دخول/خروج · موقع · مميزات.
3. **العلامات** (٣): سيزون إندونيسيا · سيزون تايلاند · سيزون المالديف (مسودة — لإثبات قابلية التوسّع) — شعار · لوحة ألوان · نمط رأس · نمط تذييل.
4. **مكتبة الوسائط** (٤٢+ أصل): رقم الأصل · الفئة · إسناد الوجهة · إسناد الفندق · علم الغلاف · علم المعرض · حالة الاعتماد.

## تجربة الإسناد (محاكاة)

`رفع صورة → إسناد إلى وجهة/فندق → غلاف/معرض → حفظ` — تُنشئ أصلاً عيّنة وتربطه فوراً، ويظهر في المعاينة مباشرةً. (محاكاة — لا رفع فعلي.)

## معاينة الدليل (الأهم) ومعاينة الفندق

تُبنى المعاينة بالكامل من محتوى الاستوديو الحالي: عدّل حقلاً (مثلاً العملة) ثم «معاينة الدليل» — يظهر التعديل فوراً. **لا محتوى ثابت في الوثيقة.** يثبت هذا مفهوم: استوديو ← دليل/فندق. إن لم تكن صورة الغلاف معتمدة يظهر تنبيه أنها لن تُستخدم في وثيقة فعلية.

## حوكمة الصور

`مسودة → مراجعة → معتمد` — تظهر بصرياً على كل أصل، مع زر تقدّم الحالة. المعتمد فقط «جاهز للاستخدام».

## أين يعيش كل شيء (معايير النجاح)

- محتوى الوجهات → قسم «محتوى الوجهات». محتوى الفنادق → «محتوى الفنادق».
- صور الدليل وصور القسيمة → **مكتبة الوسائط** (مُسندة بالوجهة/الفندق + علم الغلاف/المعرض).
- الوثائق المستقبلية تستهلك تلقائياً بالإسناد + الاعتماد — لا بحث يدوي ولا بناء يدوي.

## الربط المستقبلي (لبندر)

استبدل `window.CS_DATA` بمكتبات المحتوى وفق وثيقة Content Library Architecture (مظروف محتوى موحّد: الحالة + الإصدار + التدقيق، ومكتبة وسائط بنُسخ مقصوصة مسبقاً واعتماد بوّابي). الأصول تُشار بـ `asset_version_id` (لا حذف نهائي). الأدوار: الإدارة تدير المحتوى؛ موظفو الحجز **مستهلكون فقط**. علامات المختبر: SAMPLE DATA SOURCE · FUTURE SUPABASE DATA SOURCE · DO NOT WRITE IN LAB MODE.

**الوصول في البوابة:** الإدارة فقط (حوكمة محتوى) — الحجز والمبيعات لا يديرون المحتوى.

## تحديث Phase 14.9 — عرض التفاصيل ومحرّك المعاينة

أصبح الاستوديو **مساحة عمل لإدارة المحتوى**:

- **عرض تفاصيل الوجهة** (٨ أقسام): نظرة عامة · صورة الغلاف · المعرض · معلومات الوصول · الاتصال · العملة · الطوارئ · المناطق الموصى بها — كلها قابلة للتحرير، مع أبعاد الصورة وحالة الاعتماد وآخر تحديث.
- **لوحة «يُستخدم في»** لكل عنصر محتوى — تُظهر للإدارة أين يُستهلك (دليل · حقيبة · قسيمة · برنامج).
- **درجة جودة المحتوى /100** محسوبة من: غلاف · معرض · وصول · اتصال · عملة · طوارئ (للفنادق: غلاف · معرض · وصف · مميزات · موقع · أوقات) — لتحديد المحتوى الناقص.
- **محرّك المعاينة**: «معاينة الدليل» يُولّد دليلاً من المحتوى الحالي (غلاف · معرض · وصول · اتصال · عملة · طوارئ · مناطق موصى بها) — لا محتوى ثابت؛ تعديل أي حقل ينعكس فوراً.
- **عرض تفاصيل الفندق** بنفس المنطق + لوحة «يُستخدم في».
- **حوكمة الصور** ظاهرة على كل صورة (مسودة/مراجعة/معتمد)؛ غير المعتمد يُنبَّه عليه في المعاينة.

يثبت هذا مفهوم: **الاستوديو مصدر الحقيقة → الوثائق مخرجات تستهلك المحتوى.**

## تحديث Phase 14.95 — تجربة إدارة المحتوى (لا معاينة فقط)

تحوّل الاستوديو من معاينة إلى **تجربة إدارة فعلية (محاكاة)**:

- **إدارة الوجهة:** «تغيير صورة الغلاف» (منتقي صور مُسندة + رفع جديد) · «+ إضافة صورة للمعرض» · «×» لإزالة صورة · تحرير الوصف والوصول والشريحة والعملة والطوارئ (يُحفظ تلقائياً في الجلسة).
- **إدارة الفندق:** تغيير الغلاف · إدارة المعرض (إضافة/إزالة) · تحرير الوصف والمميزات وأوقات الدخول/الخروج والموقع.
- **إدارة مكتبة الوسائط — دورة حياة كاملة:** إضافة · استبدال · إرسال للمراجعة · اعتماد · رفض · أرشفة · استعادة. الحالات: مسودة → مراجعة → معتمد/مرفوض → مؤرشف. المعتمد فقط يظهر في الوثائق.
- **الحوكمة (تبويب جديد):** جدول الملكية والصلاحيات (من يملك/يحرّر/يعتمد) · عدّاد المحتوى المعتمد حالياً · مخطط دورة الحياة · **دليل سريع** يجيب: أين أستبدل غلاف بالي؟ أين أضيف صور أوبود؟ أين أحدّث معلومات الوجهة؟

معايير النجاح محقّقة: كل إجراء واضح وله زر/مسار صريح — الاستوديو الآن يُدار، لا يُعرض فقط.
(محاكاة فقط · لا Supabase · لا رفع فعلي · لا كتابة — تعديلات الجلسة في الذاكرة.)

## تحديث — جسر المحتوى (الاستوديو ↔ محرّك القوالب)

أصبح الاستوديو يحفظ لقطة من المحتوى في مخزن مشترك (`content-store.js`، عميل فقط) عند كل تعديل
(غلاف · معرض · حقول · دورة حياة). يقرأ **محرّك القوالب** من المخزن نفسه عند التحميل (وحيّاً إن كانا مفتوحين)،
فيظهر تغيير غلاف بالي في معاينة المحرّك فوراً — **بلا مزامنة يدوية ولا محتوى مكرّر** (مصدر واحد).
الصور المرفوعة تُصغَّر إلى نسخة خفيفة قبل التخزين. لا خادم · لا Supabase · جلسة/عميل فقط · يتلاشى بأمان إن لم يتوفر التخزين.
